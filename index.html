This is the home page for our group. It's early in the module so we haven't started anything yet. But Graeme has added some example content.

Todo list

  • All students need to create a profile page in the people folder (and linked in the people index ). Profile pages should contain the following information.

    • Who am I?
    • What experience do I have in web development?
    • What am I keen to learn from this module?
    • What kind of project would I like to be involved in?
    • A list of tentative, possibly part-formed ideas for projects. Even if they're just application names.

  • We need to discuss any ideas for proposals as a group and see which ideas feel most viable.

    Once we have ideas, we need to work in groups to create draft proposal pages (linked in proposals ) with the following information.

    • The proposed application name
    • A brief overview of the application
    • who are the users of the system?
    • Are there different types of users? e.g. administrators?
    • What data are involved?
    • Suggested user interface components

  • Once we have a few proposals in place, we can start creating teams. Students interested in working on a proposed project idea together can start a team by creating a new page. There are no limits to the size of teams, and it's not a problem if multiple teams want to build the same applicaiton.

    team pages should include:

    • A team name
    • A list of team members
    • A reference to the project you are building
    • Notes on progress
    • information about whether you are inviting new members and what skills you might need
